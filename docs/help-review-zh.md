## Overview

### RNAmeta Desktop 是什么

RNAmeta Desktop 是一个面向本地环境的 RNA 修饰峰位分析工作空间，主要用于处理 `BED` 格式的峰区间文件。它将常见 peak caller 产生的区间结果与物种对应的转录本注释信息连接起来，并将后续解释工作拆分为多个清晰的分析模块。

桌面端强调本地运行。物种选择、注释目录校验、文件上传预览、图表渲染、结果导出以及运行进程控制，都是在本地客户端内部完成的；底层分析流程由本地 R 运行环境执行。

### 当前桌面端包含的分析模块

- `Project Configuration`：配置物种和注释目录
- `Upload / Run`：上传并准备 BED 分析上下文
- `Meta Plot`：转录本相对位置分布概览
- `Peak Distribution`：峰在不同功能区域上的分布
- `Gene Statistics`：包括 `Gene Type`、`Peak Gene Size`、`Gene Matrix`
- `Exon Statistics`：包括 `Peak Exon Size`、`Peak Exon Type`、`Peak Exon Num`
- `Site`：包括 `Transcription`、`Translation`、`Splicesite`

### 与网页端的关系

桌面端与 RNAmeta 网页端使用的是同一套整体分析逻辑和模块命名，因此用户在网页端和桌面端之间切换时，概念上不会有太大割裂。网页端入口为：

https://rnainformatics.cn/RNAmeta/

不过，桌面端和网页端并不是完全相同的产品形态。网页端更适合作为项目展示、在线参考和整体入口；桌面端则更强调本地分析工作流和本地运行行为。

如果网页端与桌面端对同一模块都有说明，应优先以桌面端当前可见、可操作的行为作为实际使用依据。

### 重要范围说明

- 当前桌面端聚焦于本地分析模块。
- 网页端存在 `Motif` 工作流，但当前桌面端导航中没有这个页面。
- 如果某个功能出现在网页端而没有出现在桌面端左侧导航中，应理解为该功能当前尚未在桌面端实现。

### 桌面端推荐使用顺序

推荐工作顺序如下：

1. 配置物种和注释目录
2. 上传一个或多个 BED 文件
3. 进入具体分析模块运行分析
4. 查看图表与统计摘要
5. 导出图像或标准化表格数据

桌面端中的各个分析模块都依赖当前保存的上传上下文。如果你更换了 BED 文件，旧结果会被清空，避免图表仍然挂在错误的数据上下文上。如果你更改了物种或注释目录，已有分析结果同样会失效，因为生物学参考环境已经改变。

## Getting Started

### 第一步：选择参考物种

进入 `Project Configuration` 后，首先要在物种选择器中选定一个与注释资源匹配的物种或基因组条目。这个选择不是装饰性配置，而是后续注释目录校验和分析解释的基础。

如果你后续修改了物种，桌面端会自动清空旧分析结果。这是为了防止你继续查看在旧物种背景下生成的图表。

### 第二步：校验注释目录

选择外部注释目录后，客户端会检查该目录是否包含与当前物种对应的 `GFF`、`TxDb` 和转录本长度等必要资源。

只有当注释目录校验通过后，后续分析才具备可靠的参考背景。如果状态面板仍然提示存在缺失项，不建议继续解释后续图表结果。

#### 注释目录使用建议

- 尽量使用已经为 RNAmeta 准备好的物种专用注释目录
- 如果你在磁盘上新增或替换了文件，可以重新聚焦到客户端，让它刷新校验状态
- 在状态条明确显示注释资源完整之前，不建议进入正式分析

### 第三步：建立上传会话

进入 `Upload / Run` 页面后，上传一个或多个 BED 文件。客户端会对上传内容进行规范化处理，并建立当前会话上下文，供后续各个分析模块复用。

这意味着你不需要在每个分析模块里重新选择输入文件。桌面端使用的是“一个上传上下文，多个分析视图”的方式。

### 第四步：运行模块并导出结果

上传上下文准备完成后，可以进入任意分析模块进行运行。每个模块通常会输出图表区、统计摘要以及导出入口。

图像导出适合论文、汇报和展示；数据导出适合进一步统计分析、表格归档和外部重新作图。

## Upload / Run

### Upload / Run 的作用

`Upload / Run` 是整个桌面端分析流程的共享入口。它负责统一准备 BED 输入数据，而不是让每个模块各自再去单独选文件。

这样做的好处是工作流更加稳定：一套上传上下文，可以被多个分析模块重复使用，也更容易确认当前会话中究竟分析的是哪一组文件。

### BED 文件的使用方式

上传的是你希望分析的峰区间 BED 文件。在多文件场景下，文件名会继续在图例、tooltip 和导出数据中出现，因此建议文件命名尽量清晰、可辨认。

桌面端会先对 BED 内容做一次本地规范化处理，随后分析模块读取的是规范化后的表示形式。但对用户来说，真正的数据上下文依然是你上传的原始 BED 文件。

#### 推荐习惯

- 同一次分析会话中尽量只放同一物种、同一注释背景下的数据
- 尽量使用具有区分度的文件名
- 如果你要切换到另一组样本，请直接重新上传，而不是延续旧结果做解释

### 文件变化后会发生什么

如果你重新上传了新的 BED 集合，桌面端会清空旧模块结果。这样可以防止页面上还残留上一轮样本的图和摘要。

如果你重复上传的是同一批原始 BED 文件，桌面端可以在当前应用会话中更稳定地复用缓存，从而减少重复计算。

### 为什么 Upload / Run 很关键

后续的 `Meta Plot`、`Peak Distribution`、`Gene Statistics`、`Exon Statistics`、`Site` 模块，都会依赖这里建立的上传上下文。

如果这里的数据没有确认清楚，后面即使图画出来了，也可能不值得解释。

## Meta Plot

`Meta Plot` 用于生成转录本相对位置上的总体分布曲线，适合观察峰信号沿标准化转录本结构的变化趋势。

它适合回答的问题包括：信号是否在某类转录本边界附近富集，不同样本是否表现出相似的整体分布轮廓，以及调整参数后曲线形状是否发生明显变化。

在使用时，应先配置好相关控制参数，再运行分析。结果出来后，建议同时查看曲线图和统计摘要，而不要只看其中之一。

如果你需要继续在外部绘图或做进一步统计，可以通过导出面板把当前图背后的标准化表格导出为 `CSV` 或 `TXT`。

## Peak Distribution

`Peak Distribution` 用于将上传的峰区间注释到不同基因功能区域中，例如启动子、UTR、CDS、内含子和基因间区等。

它最适合回答的问题是：当前峰集在基因结构的哪些区域更常见，不同样本之间在功能区域分布上是否存在明显差异。

控制面板允许你选择当前图中显示哪些 feature。需要注意的是，导出的数据会跟随当前可见的图表状态，而不是总是导出一整套隐藏与可见并存的类别。

如果结果看起来异常，第一步应先检查 `Project Configuration` 中的物种和注释目录是否与当前数据真正匹配。

## Gene Statistics

`Gene Statistics` 由三个部分组成：`Gene Type`、`Peak Gene Size`、`Gene Matrix`。

### Gene Type

`Gene Type` 用于总结与峰关联的转录本生物类型，例如是否主要富集在蛋白编码基因，还是在其他非编码类型中更常见。

它适合在分析早期快速建立“这批峰主要落在哪类转录本上”的整体印象。

### Peak Gene Size

`Peak Gene Size` 用于观察与峰相关联的转录本长度分布。它适合回答：当前信号是否主要偏向较长或较短的转录本，不同样本是否具有不同的长度分布特征。

由于这是一个分布型结果，建议结合图形形态和摘要数值一起看。如果需要进一步做外部统计比较，可以导出表格数据。

### Gene Matrix

`Gene Matrix` 是多文件交集关系模块，适用于 2 到 5 个上传文件之间的基因重叠比较。

它特别适合用来判断样本之间的共有基因覆盖情况，例如重复样本之间是否高度一致，或者不同处理组之间是否呈现分离趋势。

## Exon Statistics

`Exon Statistics` 包括 `Peak Exon Size`、`Peak Exon Type`、`Peak Exon Num` 三个模块，主要从外显子结构角度解释当前峰集相关联的转录本特征。

### Peak Exon Size

`Peak Exon Size` 用于总结外显子长度分布，并保留首个、中间和末端外显子之间的区分。

如果你怀疑信号与某类外显子长度结构有关，这个模块会很有价值。

### Peak Exon Type

`Peak Exon Type` 强调的是外显子位置构成，而不是长度本身。它用于比较峰更多出现在首个外显子、中间外显子还是末端外显子。

这是一个更偏组成型的视图，通常适合快速阅读和高层次总结。

### Peak Exon Num

`Peak Exon Num` 用于总结与峰关联转录本的外显子数量分布，帮助判断当前信号是否更偏向结构简单还是结构复杂的转录本。

这个模块适合和 `Peak Exon Size` 一起看，从长度和复杂度两个角度补充结构解释。

## Site

`Site` 分组包含 `Transcription`、`Translation` 和 `Splicesite` 三个边界位点相关模块，用于分析信号在特定生物学边界附近的局部分布。

### Transcription

`Transcription` 关注的是转录相关边界附近的峰密度，例如转录起始和终止邻域。它适合回答是否在转录边界附近存在局部富集。

桌面端中，这类模块通常同时提供密度曲线和热图视图，两者应配合阅读。

### Translation

`Translation` 关注的是翻译相关边界，例如起始密码子和终止密码子邻域。适合那些更关心编码区出入口附近富集模式的分析场景。

### Splicesite

`Splicesite` 关注的是剪接位点邻域，包括 5' 和 3' 剪接相关区域。它是 Site 组里最偏局部结构的问题导向模块。

如果热图或曲线形状异常，建议先重新核对上传文件、物种和注释目录是否处于同一生物学参考框架下。

## Export & Cache

### 图像导出

每个分析模块都带有导出面板。图像导出适合生成用于论文、报告、幻灯片或补充材料的图。

如果你的目标是保留桌面端当前图表的视觉状态，应优先使用图像导出。

### 数据导出

当前桌面端也支持将图表背后的标准化数据导出为表格：

- `CSV`：逗号分隔
- `TXT`：制表符分隔

这些导出文件只包含数据本身，不会附带叙述性 summary 文本，因此更适合机器读取、再作图和统计分析。

### 会话缓存

桌面端会在当前应用会话中使用临时缓存，以提升重复分析速度。缓存位于应用自己的本地缓存目录，而不在项目目录中。

这个缓存只用于加速，不适合作为长期结果存储目录。真正需要长期保留的结果，应通过导出面板保存到你指定的位置。

### 控制台与进程控制

底部控制台用于显示运行日志和进程状态。当分析正在运行时，可以在控制台中使用终止按钮中断当前本地运行进程。

如果你主动终止运行，页面中可能会短暂出现 `Aborted` 状态，然后自动消失，这是正常行为。

## FAQ / Troubleshooting

### 为什么我改了东西之后看起来像还是旧结果

如果你重新上传了 BED 文件，客户端会清空旧结果；如果你修改了物种或注释目录，旧结果也会自动失效。这是为了防止你继续读取错误背景下生成的图。

如果页面仍然看起来不一致，最稳妥的做法是重新确认 `Project Configuration` 和 `Upload / Run`，然后重新运行目标模块。

### 为什么注释目录总是校验不过

最常见原因是：当前物种选择与目录中的注释资源不匹配，或者目录中缺少某些必须文件。

只有当所选物种能够在该目录中找到对应的注释资源时，桌面端才会认为该目录有效。

### 为什么网页端有的功能桌面端没有

网页端和桌面端属于同一项目体系，但并不是功能完全一致的两个外壳。网页端可以作为项目背景和在线参考入口，但桌面端只包含当前本地客户端已实现的部分。

例如，网页端存在 `Motif` 工作流，但它当前并不在桌面端导航中。

### 为什么重复分析会变快，那些文件到底存在哪里

重复分析加速主要来自桌面端的会话缓存。缓存文件位于应用缓存目录，而不是项目文件夹中，也不是用户应当长期管理的正式输出目录。

如果你需要长期保留分析结果，请使用导出功能，而不要依赖缓存。
